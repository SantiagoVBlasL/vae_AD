03_b — Transfer v1 Audit & Visualisation¶

Purpose: Post-hoc audit of the three transfer-learning variants (V1-A raw-edges, V1-B frozen-encoder probe, V1-C unsupervised recon-adapt + probe) run by scripts/run_transfer_v1_three_variants.py.

What the scores mean: Each variant trains a logistic classifier on ADNI AD vs CN and then applies that classifier to the COVID cohort. The resulting y_score is an "AD-likeness" probability — not a disease-severity or diagnostic score.

Sections:

  1. Setup & helpers
  2. Manifest audit

  5. COVID score distributions by Grupo and Sex

  7. ROC: Grupo discrimination using AD-likeness scores
  8. Optional: MOCA correlations

We normalise both sides (strip, upper, remove whitespace) and merge on the ID column.

8b — COVID vs CONTROL Classification: Per-ADNI-Fold Latent Features¶

The Latent Misalignment Problem (Early Fusion ≡ Invalid)¶

The original v1 pipeline extracted latent $\mu$ from each of the 5 ADNI-fold VAEs and averaged them element-wise before feeding them to the classifier. This is mathematically invalid.

Reason: VAEs trained independently on different folds are not identifiable. Their latent dimensions are not permutation-aligned — dimension $j$ in $\text{VAE}_{\text{fold}=1}$ may encode a completely different connectivity pattern than dimension $j$ in $\text{VAE}_{\text{fold}=2}$. Averaging $\mu_{\text{fold}=1}$ with $\mu_{\text{fold}=2}$ therefore destroys the topological structure of each individual representation.

Solution: Late Fusion¶

We run the full nested-CV classification pipeline independently on each ADNI-fold's latent representation, producing 5 separate sets of out-of-fold predictions. Only at the very end do we ensemble the predicted probabilities (Late Fusion).

The plot below shows the AUC distribution across the 5 ADNI encoder folds, grouped by classifier type, using the results from scripts/run_covid_vs_control_v1.py.

COVID_CLF_DIR = PROJECT_ROOT / "results" / "covid_vs_control_v1"

    # Ensure columns exist (v2 adds feature_family + adni_encoder_fold)
